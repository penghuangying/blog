instanceof操作符的问题在于，它假定单一的全局执行环境。如果网页中包涵多个框架，那实际上就存在两个以上不同的全局执行环境，从而存在两个以上不同版本的 Array 构造函数。如果你从一个框架向另一个框架传入一个数组，那么传入的数组与在第二个框架中原生创建的数组分别具有各自不同的构造函数。 ES5 开始，用 Array.isArray() 方法可以解决这个问题。

好记性不如烂笔头

发表于 2016-07-01   |  
日常工作和生活中碰到的技术性的，非技术性的，各种问题，记下来，以便今后翻阅的，持续更新中…

1.文件在线预览

  一开始不知道文件在线预览要怎么实现，于是，就先看看其它人是怎么实现的。看了 coding.net 的，发现其预览 office 文件用的是微软的Office Web Apps Viewer。google 了之后也基本上只有 Google Docs Viewer 和 微软的这个能复合我的使用场景了。使用方法如下：

Google Docs Viewer

https://docs.google.com/viewer?url=[FILE_URL]

Office Web Apps Viewer

https://view.officeapps.live.com/op/view.aspx?src=[FILE_URL]

  按照上面的方法生成文件预览的 url 后，有 iframe 嵌入到自己网站的页面中就可以预览了。因为墙的原因，所以放弃了谷歌的，但微软的是只支持 office 文件，PDF 都不支持，要预览 PDF 文件怎么办呢？直接用浏览器打开预览就可以了，iframe 的 src 为 PDF 文件的原路径久可以咯。当时还碰到一个问题是，一开始后端给的文件路径名是包含端口号的，类似为 https://site.com:9090/xxx/xxx.docx ; 不知道为甚么当路径名中存在端口号时，微软的 Viewer 不能获取到这个文件还是怎么回事，要么一直停留在 fetch 的页面，或者直接报个 error 页面出来。最后还是通过修改 nginx 配置来去掉端口号，最后实现了文件在线预览的功能。但预览的速度实在是有点慢。
