LazyMan(“Hank”).sleep(10).eat(“dinner”)输出

Hi! This is Hank!

LazyMan(“Hank”).eat(“dinner”).eat(“supper”)输出

我试了一下，把 filePreview directive 里的 iframe 标签加了个 sandbox='' :

<iframe ng-if="file_type != 'default' && file_type!= 'image'" src="result_file_url" sandbox=''></iframe>

嗯，果然已经没有这些 JS 错误了,可又发现，这样一来，文件预览功能又不行了，这个 iframe 加载不了内容了，继续看相关的文档，iframe.aspx ) 元素, 和 security.aspx ) 属性，security 属性说明如下：

最后，iframe 标签如下：

<iframe ng-if="file_type != 'default' && file_type!= 'image'" src="" security="restricted"></iframe>

不再报 JS 错误，文件预览功能也不受影响，至此，也算是解决这个问题了。从发现问题，不断猜想和去验证猜想，debug 前前后后花了我一天的时间，最后解决问题仅仅是加了个 security="restricted" ，哎，真的是有点哭笑不得的样子。但程序员嘛，总是在不断解决问题中总结，成长。最后如果能顺利解决问题，自然会多一分成就感的喜悦，最后就算未能解决问题，在搜索解决相关问题的过程中，也会阅读到很多相关文档，学到很多知识，过程自然也是愉悦的。最后我想说，以代码量来衡量一个程序员的能力与价值的话，那纯属扯淡！哈哈！

instanceof操作符的问题在于，它假定单一的全局执行环境。如果网页中包涵多个框架，那实际上就存在两个以上不同的全局执行环境，从而存在两个以上不同版本的 Array 构造函数。如果你从一个框架向另一个框架传入一个数组，那么传入的数组与在第二个框架中原生创建的数组分别具有各自不同的构造函数。 ES5 开始，用 Array.isArray() 方法可以解决这个问题。

5. slice , subtring, substr 方法比较:

这 3 个方法都接受 2 个参数，第一个都表示起始位置(包含)，slice 和 substring 接受的第二个参数表示结束位置(不包含)，substr 第二个参数表示截取的子串长度。若没有第二个参数，3 个都表示截取到末尾；若参数有负数，slice 方法是将负数加上字符串长度代替； substring 是设为 0； substr 则是将第一个参数加上长度处理，第二个设为0；若第二个参数小于第一个参数，slice 返回空串，substring 则是调换参数位置。
