对于一个网页，或者一个全局作用域而言，使用 instanceof 操作符就可以：

instanceof操作符的问题在于，它假定单一的全局执行环境。如果网页中包涵多个框架，那实际上就存在两个以上不同的全局执行环境，从而存在两个以上不同版本的 Array 构造函数。如果你从一个框架向另一个框架传入一个数组，那么传入的数组与在第二个框架中原生创建的数组分别具有各自不同的构造函数。 ES5 开始，用 Array.isArray() 方法可以解决这个问题。

4. 数组和字符串的一些方法：

数组中 a. splice() 方法总是返回一个数组，包含了被删除项。 b.迭代方法：every,some 返回布尔值；filter 返回该函数会返回 true 的项组成的数组；forEach 没有返回值；map 返回每次函数调用结果组成的数组。 c.缩小方法：reduce() 和 reduceright():

字符串中： a.slice(start,end) 和 substring(start, end)第二个参数表示结束位置,而且 substring 方法会自动将较大参数为结束位置，即 substring(0,3)和substring(3,0)是一样的; 而 substr(start, len)第二个参数表示截取长度。若给的参数为负，slice() 方法会将传入的负值与字符串长度相加， substring() 会把所有负值参数都转为 0； substr() 将第一个负的参数与字符串长度相加，第二个参数转为 0；b.trim（） 方法会创建一个字符串副本，删除前置和后缀所有空格。

4. slice , subtring, substr 方法比较:

这 3 个方法都接受 2 个参数，第一个都表示起始位置(包含)，slice 和 substring 接受的第二个参数表示结束位置(不包含)，substr 第二个参数表示截取的子串长度。若没有第二个参数，3 个都表示截取到末尾；若参数有负数，slice 方法是将负数加上字符串长度代替； substring 是设为 0； substr 则是将第一个参数加上长度处理，第二个设为0；若第二个参数小于第一个参数，slice 返回空串，substring 则是调换参数位置。
